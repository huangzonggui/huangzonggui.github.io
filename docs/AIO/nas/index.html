安装群晖 Nas​

共需要两个包：下载 arpl 镜像和群晖对应版本的 pat 补丁

  1. 通过镜像 arpl 编译引导
  2. 编译完启动安装，选择 pat 补丁
  • esxi 遇到安装失败，无法格式化磁盘

    解决：群晖版本换为 920+

参考：

  • https://www.tenmeng.com/archives/10.html

开启 docker ipv6​

手动创建（配置 ipv6: true, 重启 docker 会消失，为何？）

docker network create --driver bridge --ipv6 --subnet "fd00:dead:beef::/64" apisix-docker_apisix

network 命名规则： project名称_配置文件中的network名称

运行 docker-compose 会自动创建network

docker-compose -p apisix-docker up -d

查看network

docker network ls

查看network详情

docker network inspect apisix-docker_net

群晖 nas 设置 root 登录​

  • 配置 sshd
  • 重启 sshd
  • sudo synouser --setpw root password

群晖Non-Root非特权账户执行Docker指令​

sock 用于命令和docker守护进程通信

修改套接字的 /var/run/docker.sock 组所有权

root@nas:~# ll /var/run/docker.sock 
srw-rw---- 1 root root 0 Aug 23 12:52 /var/run/docker.sock

root@nas:~# sudo synogroup --enum
No Enumerate agrument is specified.
Assumeing AUTH_ALL.
3 Group Listed:
administrators
http
users

root@nas:~# sudo chgrp administrators /var/run/docker.sock

root@nas:~# ll /var/run/docker.sock 
srw-rw---- 1 root administrators 0 Aug 23 12:52 /var/run/docker.sock
  • https://www.simaek.com/archives/467/

问题：重启 docker 后失效

临时解决方案: 单个容器代理

docker run -d -p 8081:8081 -v /volume2/share/downloads:/downloads \ -e HTTP_PROXY="http://192.168.3.43:7890" \ -e HTTPS_PROXY="http://192.168.3.43:7890" \ ghcr.io/alexta69/metube
